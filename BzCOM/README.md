# BzCOM

Qt-based GUI 通信工具，支持串口（UART）和 TCP 网络通信，可用于嵌入式开发调试、OTA 升级等场景。

## 功能特性

### 通信模式
- **UART 串口**：支持配置波特率、数据位、校验位（无/奇/偶）、停止位（1/2）
- **TCP 客户端**：主动连接服务器
- **TCP 服务器**：监听端口，等待客户端连接

### 数据收发
- **文本模式**：可读字符串发送
- **Hex 模式**：十六进制格式发送（如 `FF 01 02`）
- **发送历史**：快捷发送常用数据

### 文件传输
- 基于 ACK 的重传机制
- 256 字节分包传输
- 超时自动重试（默认 1 秒超时）

### 辅助功能
- **校验和计算**：支持 Xor、Sum、CRC8、CRC16、CRC32
- **通信日志**：实时显示收发数据，支持保存和清除
- **OTA 控制器**：与下位机 OTA 协议对接，支持固件升级

## 构建

### 环境要求
- Qt 5.15+ (MinGW 64-bit 或 MSVC)
- C++11
- Qt 模块：widgets、serialport、network

### 编译步骤

```bash
# 进入项目目录
cd BzCOM

# 生成构建文件
qmake BzCOM.pro

# 编译 (Windows MinGW)
mingw32-make

# 或使用 Qt Creator 直接打开 .pro 文件构建
```

### 运行
编译完成后，在 `build/` 目录生成 `BzCOM.exe`，双击运行即可。

## 使用说明

### 串口模式
1. 选择 "UART" 模式
2. 从下拉列表选择串口号
3. 配置波特率、数据位、校验位、停止位
4. 点击 "打开串口"
5. 发送数据或进行文件传输

### TCP 模式
1. 选择 "TCP" 模式
2. 选择 "客户端" 或 "服务器" 角色
3. 客户端：填写服务器 IP 和端口，点击连接
4. 服务器：填写监听端口，点击开始监听
5. 等待连接建立后收发数据

### Hex 模式
- 发送时输入格式：`FF 01 02 03` 或 `ff010203`
- 自动过滤空格和无效字符

### OTA 升级
1. 点击 "文件传输" 按钮
2. 选择要传输的固件文件（.hex/.bin）
3. 等待传输完成

## 通信协议

### OTA 包格式
```
[包头 2B] [长度 2B] [类型 1B] [数据 N] [校验 2B]
```

### ACK 机制
- 接收方收到正确包后回复 ACK
- 超时未收到 ACK 发送方自动重传
- 最大重试次数可配置
