🔌外设配置

外设 引脚 说明
USART1 PA9=TX, PA10=RX 与上位机通信，115200bps, 8N1

AT24C64 EEPROM 存储

地址 内容 说明
0x10 启动模式 MODE_BOOT_UPDATE / NO_UPDATE / RESET / UART_IAP / OTA
0x11-0x12 密钥 必须为 0x5A6B 才生效，防止误操作
0x13 活跃分区 0xAA = APP_A, 0xBB = APP_B

🚀启动模式

模式值 名称 说明
0x01 MODE_BOOT_UPDATE W25Q64 中已有固件，执行升级
0x02 MODE_BOOT_NO_UPDATE 不更新，直接启动
0x03 MODE_BOOT_RESET 恢复出厂设置
0x04 MODE_BOOT_UART_IAP 串口 IAP，直接写入目标分区
0x05 MODE_BOOT_OTA_SELECT OTA 模式选择（按 KEY1 后的握手阶段）
0x07 MODE_BOOT_OTA UART OTA，串口→W25Q64→目标分区

check_update()

从 AT24C64 读取启动模式，校验密钥 (0x5A6B)，确定活跃分区和目标分区。

check_default()

5秒内等待用户按键选择模式，支持 KEY1 (OTA) 和 KEY2 (UART IAP)，带 50ms 消抖。

update()

根据模式执行升级：W25Q64 升级、UART IAP 或 OTA 接收。

copy_and_jump()

将活跃分区固件复制到 APP_RUN 区域，校验栈顶地址后跳转到应用程序。

🔐双区备份机制

两个固件分区 (A/B) 互为备份，升级时写入非活跃分区，校验通过后切换。确保升级失败时可回滚。

📦 APP_B 分区

固件校验

跳转前校验固件有效性：检查栈顶地址必须在 SRAM 范围内 (0x20000000 - 0x20005000)。

uint8_t is_valid_firmware(uint32_t addr)
{
    uint32_t stack = *(volatile uint32_t *)addr;
    return (stack >= SRAM_STACK_BASE && stack <= SRAM_MAX);
}

🎮按键选择流程

复位后 5 秒内按下按键，决定进入哪种升级模式。

KEY1
OTA 模式选择
  1. 按下 KEY1 (PB13)
  2. 进入 OTA_SELECT 模式
  3. 等待上位机握手
  4. 发送 '1' → UART OTA
  5. 发送 '2' → 保留扩展
KEY2
UART IAP 模式
  1. 按下 KEY2 (PB12)
  2. 进入 UART_IAP 模式
  3. 等待启动字符 '1'
  4. 串口接收 bin 数据
  5. 直接写入目标分区

📡OTA 通信协议

基于 Modbus CRC16 的自定义协议帧，支持固件分片传输、校验和应答。

协议帧格式

0x55AA
帧头
2 字节
CMD
命令码
2 字节
LEN
数据长度
2 字节
DATA
数据区
N 字节
RSV
保留
4 字节
CRC
校验
2 字节

命令码

命令 值 说明
CMD_UPDATE_START 0x0003 升级开始，DATA[0-3] 为固件大小
CMD_UPDATE_DATA 0x0004 升级数据，DATA[0-3] 为 offset, DATA[4:] 为数据
CMD_UPDATE_END 0x0005 升级结束
CMD_ACK cmd | 0x0080 应答成功
CMD_NAK cmd | 0x00FF 应答失败

📥OTA 完整流程

1
按 KEY1 进入 OTA 选择模式
系统输出菜单，等待上位机选择通信方式
OTA_MODE: 1=UART 2=保留
2
上位机发送 '1' 选择 UART OTA
下位机进入 OTA_Receive()，发送 OTA_START，开始擦除 W25Q64
进入UART_OTA OTA_START
3
协议帧传输 (CMD_UPDATE_START)
上位机发送固件大小，下位机回复 ACK
| 0x55AA | 0x0003 | 0x0400 | size[4] | 0x0000 | CRC16 | ← ACK (0x5583)
4
协议帧传输 (CMD_UPDATE_DATA)
分片传输固件数据，每包最大 258 字节，下位机写入 W25Q64
| 0x55AA | 0x0004 | 0xnn04 | offset[4] | data[N] | CRC16 | ← ACK (0x5584)
5
传输完成 (CMD_UPDATE_END)
下位机保存元数据，擦除目标分区，复制 W25Q64 → Flash
| 0x55AA | 0x0005 | 0x0000 | 0x0000 | CRC16 | ← UPDATE_OK 或 INVALID_FW
6
分区切换与跳转
校验固件有效后，切换活跃分区到新固件，下次启动运行新固件
AT24C64[0x13] = 0xBB (切换到 APP_B)
